Built for engineers who want to understand the hardware

Most engineers learn GPU programming backwards — they add torch.compile and hope for the best. This course goes the other direction. Every technique is derived from hardware constraints you can calculate yourself. The roofline model explains when fusion helps. The memory hierarchy explains why tiling exists. The bit representation explains when BF16 is safe and when it isn't. No cargo-culting. Every optimization is a consequence you derive, not a rule you memorize.
